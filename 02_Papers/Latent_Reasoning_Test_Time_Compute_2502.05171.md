## 1. 도입 및 배경 (Introduction)
* **논문 요약 및 배경**:
  - 기존의 테스트타임 연산(Test-time compute) 확장 방식은 주로 긴 Chain-of-Thought(CoT)를 통해 생성 토큰 수를 늘리는 것에 의존함.
  - 이 방식은 길고 복잡한 데모 데이터를 필요로 하며, 텍스트 형태의 긴 컨텍스트를 유지해야 하는 비효율성이 존재함.
  - 본 논문은 토큰을 늘리는 대신, **연속적인 잠재공간(Latent Space)에서 동일한 블록을 반복(Recurrent) 실행**하는 것만으로 대규모 사전학습 모델의 추론 성능을 향상시킬 수 있는지 탐구함.

* **모델 진화/비교 (표 활용)**:

| 비교 항목 | 기존 Test-time Compute (CoT 기반) | 제안 모델 (Latent Reasoning) |
|---|---|---|
| **연산 확장 방식** | 텍스트(토큰) 생성을 길게 늘려 연산량 증가 | 잠재공간에서 동일한 블록을 여러 번 반복(Unroll) 실행 |
| **데이터 요구사항** | 길고 복잡한 CoT 데모 데이터 필수 | 긴 CoT 데모 없이 일반적인 사전학습 데이터 활용 가능 |
| **효율성** | 긴 컨텍스트 처리 및 저장 부담 큼 | 텍스트를 생성하지 않고 벡터 공간에서 처리하여 컨텍스트 부담 감소 |

* **핵심 개선점/기여도 (표 활용)**:

| 핵심 기여 | 설명 |
|---|---|
| **잠재공간 반복 추론** | 토큰 생성 없이 디코더의 특정 코어 블록을 잠재공간에서 여러 번 반복하여 연산량을 스케일링하는 축 확립 |
| **효율적인 대규모 학습** | 로그정규-푸아송 분포로 반복 횟수를 샘플링하고 마지막 \(k\) 스텝만 역전파하는 방식으로 800B 대규모 학습 성공 |
| **직관을 넘어서는 잠재 표현** | 언어로 표현하기 애매한 복잡한 추론 과정을 벡터 수준에서 내재화하여 처리할 수 있는 가능성 제시 |

## 2. 아키텍처 및 훈련 전략 (Architecture & Training)
* **아키텍처 구조 (표 활용)**:

| 구성 요소 | 역할 및 특징 |
|---|---|
| **Prelude (P)** | 입력 토큰을 받아 초기 임베딩 \(e = P(x)\)을 생성하는 인코딩 파트 |
| **Recurrent Core (R)** | \(s_i = R(e, s_{i-1})\) 구조로 동일한 가중치를 공유하며, 잠재공간에서 지정된 횟수만큼 반복 연산을 수행하는 핵심 블록 |
| **Coda (C)** | 반복 연산이 끝난 최종 상태 \(s_r\)을 받아 다음 토큰의 확률 \(p = C(s_r)\)을 출력하는 디코딩 파트 |

* **훈련 과정 (표 활용)**:

| 단계 및 기법 | 설명 |
|---|---|
| **반복 횟수 샘플링** | 매 학습 스텝마다 반복 횟수 \(r\)을 '로그정규-푸아송' 분포에서 무작위로 뽑아 기대 손실을 최적화함 |
| **메모리 절약 (Truncated Backprop)** | 메모리 절약을 위해 모든 반복 스텝을 저장하지 않고, 마지막 \(k\) 스텝(본문 \(k=8\))에 대해서만 역전파(Backpropagation) 수행 |
| **구현 및 안정화 디테일** | 코어 블록에서 \(s\)와 \(e\)를 Concat 후 Adapter를 통해 안정화하며, 큰 모델에서는 Sandwich-style Norm이 필수적으로 적용됨 |

## 3. 실험 및 결과 (Experiments)
* **구현 디테일 (표 활용)**:

| 항목 | 디테일 |
|---|---|
| **모델 체급** | 3.5B 파라미터 규모의 잠재 반복형(Latent Recurrent) 모델 |
| **학습 데이터** | 800B 토큰 규모 (코드, 수학, 과학 비중이 높은 공개 데이터 혼합 + 지시 데이터 일부 포함) |
| **학습 설정** | 시퀀스 길이 4096, BPE 크기 65536 적용 |

* **벤치마크 결과 (SOTA 달성) (표 활용)**:

| 평가 항목 | 결과 요약 |
|---|---|
| **수학 및 코딩 추론** | 공개 데이터 기준 Pythia 모델보다 우수하며, 비슷한 설정의 비반복(Non-recurrent) 모델과 비교 시 고난도 과제에서 뚜렷한 성능 이득 확인 |
| **반복(r) 증가에 따른 성능** | 테스트타임에 반복 횟수 \(r\)만 키워도 GSM8K CoT 및 HumanEval 성적이 지속적으로 상승함 (예: GSM8K CoT flexible ~47.23%) |
| **과제별 포화 속도 차이** | HellaSwag 같은 직관적인 과제는 적은 반복 횟수(~8번)에서 빠르게 포화되나, GSM8K 등 복잡한 추론 과제는 더 많은 반복 시 성능 향상 지속 |

* **정성적 결과 (Qualitative Results)**:
  - 사전학습 데이터만으로도 복잡한 수학 및 코딩 문제에서 반복 연산을 통한 성능 이득을 확인했으며, CoT에 전적으로 의존하지 않는 문제 해결 과정을 보임.

* **절제 연구 (Ablation Studies)**:
  - 잠재 공간의 궤적(Trajectory)을 PCA로 분석하여, 토큰마다 잠재 상태가 특정 점으로 수렴하거나, 궤도(Orbit)를 돌거나, 슬라이더처럼 움직이는 다양한 역학적 특성을 가짐을 밝힘. 이를 산술 연산이나 복잡한 숙고 과정과 연관 지어 해석함.

## 4. 요약 및 시사점 (Conclusion)
* **요약**:
  - 긴 텍스트(Token) 생성에 의존하지 않고, 모델의 깊이를 잠재공간 내에서 반복(Unroll)시키는 것만으로 테스트타임 연산을 스케일링할 수 있음을 성공적으로 입증함.
* **한계점 (Limitations)**:
  - 현재 3.5B 스케일의 개념 증명(PoC) 수준으로, 더 큰 스케일로의 확장, 최적의 데이터 믹스, 분산 학습 환경에서의 하드웨어 및 샘플링 최적화 등 기술적 고도화 여지가 많음.
* **시사점 및 활용 방안**:
  - 향후 포스트트레이닝 기법을 통해 반복 연산을 효율적으로 압축(Distillation)하거나, 강화학습(RL)을 결합하여 난이도를 제어하는 등 확장 가능성이 매우 큼.
  - 기존의 CoT 방식을 잠재 순환 구조 내로 내재화(Internalize)하거나 MoE(Mixture of Experts)와 결합하여 차세대 에이전틱 모델의 연산 효율성을 극대화할 수 있는 방향을 제시함.

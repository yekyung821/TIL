## 1. 도입 및 배경 (Introduction)

### 1.1 모델 진화 요약
기존 다중모달 모델들의 한계(이해/생성의 성능 타협)를 극복했던 **Janus**의 아키텍처를 계승하되, 스케일업을 통해 성능을 SOTA급으로 폭발시킨 후속작.

| 구분 | 모델 특징 | 극복한 한계 / 개선점 |
| :---: | :--- | :--- |
| **기존 통합 모델** | 시각 이해/생성에 **단일 인코더** 사용 | 요구사항 충돌로 인한 성능 타협(Trade-off) 발생 |
| **Janus** (전작) | 이해/생성 인코더를 분리 (**Decoupling**) | 타협 없이 두 기능 모두 전용 모델 수준 달성 |
| **Janus-Pro** (본작) | 디커플링 유지 + **대규모 스케일링** | 최적화 훈련, 데이터 확장, 파라미터 7B 확장 |

### 1.2 Janus-Pro의 3대 핵심 개선점
| 개선 요소 | 주요 변경 사항 | 기대 효과 |
| :---: | :--- | :--- |
| **훈련 전략** | 훈련 스텝(Steps) 대폭 증가 및 비율 세밀 조정 | 모델의 안정적 학습 및 수렴 |
| **데이터 양/질** | 실제 데이터 + 고품질 합성(Synthetic) 데이터 | 데이터 다양성 확보 및 프롬프트 준수력 상승 |
| **모델 체급** | 기존 1B ➔ **7B (DeepSeek-LLM 7B 기반)** | 언어적 추론력 및 전반적 퍼포먼스 극대화 |

## 2. 아키텍처 및 훈련 전략 (Architecture & Training)

### 2.1 아키텍처: 디커플링 투트랙(Two-track) 비교
이해와 생성, 두 마리 토끼를 잡기 위해 서로 다른 최적의 도구(인코더)를 사용함. 두 경로는 최종적으로 단일 **자기회귀(Autoregressive) LLM**에서 통합 처리됨.

| 경로 (Path) | 시각 인코더 | 처리 방식 | 추출 정보 특징 |
| :---: | :---: | :--- | :--- |
| **이해 (Understanding)** | **SigLIP** | 이미지 ➔ 2단 MLP 어댑터 ➔ LLM 텍스트 공간 매핑 | 고차원적이고 추상적인 **의미(Semantic)** 정보 |
| **생성 (Generation)** | **VQ Tokenizer** | 이미지 ➔ 1D 이산형 토큰(ID) 변환 | 공간적이고 세밀한 **픽셀/구조** 정보 |

*Figure 2 | 이해와 생성을 완벽히 분리한 디커플링 아키텍처*

### 2.2 3단계 훈련 과정 (Training Stages)
스케일업된 데이터와 파라미터를 감당하기 위해 더욱 빡세게 다듬어진 3단계 훈련.

| Stage | 훈련 단계명 | 상태 (Freeze/Unfreeze) | 주요 목표 및 특징 |
| :---: | :---: | :--- | :--- |
| **1** | **어댑터 훈련** | LLM(🔒), 시각 인코더(🔒) | 시각-언어의 기초 연결 (어댑터만 가볍게 훈련) |
| **2** | **통합 사전 훈련** | **전체 락 해제 (🔓)** | 대규모 텍스트+이미지 혼합 학습. 훈련 스텝 크게 증가. |
| **3** | **지도 미세 조정**<br>(SFT) | **전체 락 해제 (🔓)** | 유저 지시(Prompt) 대응력 튜닝. 고품질 합성 데이터 대거 투입. |

## 4. 요약 및 시사점 (Conclusion)

* **Scaling Law의 위력 증명**
  * 완벽한 아키텍처(`디커플링`)에 압도적 체급(`7B & 대규모 데이터`)이 더해졌을 때 일어나는 시너지를 입증함.
* **타협 없는 범용 멀티모달의 시대**
  * '보는 것(이해)'과 '그리는 것(생성)' 사이의 성능 줄다리기를 끝냄.
  * 차세대 **범용 멀티모달 AI (Unified Multimodal Model)**가 나아가야 할 가장 확실하고 모범적인 해답(Milestone)을 제시함.

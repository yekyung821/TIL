### ⭐ 6.1 [IR] 과학지식 질문 질의응답 ― 과천과학관 직무와 가장 직결되는 프로젝트

> 과학 상식 질의에 대해 **질의 의도 판별 → Sparse/Dense 하이브리드 검색 → RRF → 2-stage Reranking** 으로 **MAP 0.8970** 을 달성한 IR·RAG 전단 프로젝트.

| 항목 | 내용 |
|---|---|
| 기간 | 2025.12.18 ~ 2025.12.29 |
| 팀 | 서치스파크 (팀원) |
| 최종 MAP | **0.8970** |
| 기술 스택 | Python 3.10, **Elasticsearch 8.x**, KURE-v1 / multilingual-e5-large-instruct 임베딩, **BGE-reranker-v2-m3** 앙상블, GPT-4o-mini, Custom RAG pipeline |
| 핵심 파이프라인 | Sparse (BM25) + Dense 병렬 검색 → Multi-query (original / expanded / conceptual) → **RRF** 통합 → 2-stage Reranking |
| 데이터 규모 | 과학 문서 3,849 + 비과학 423 (총 4,272건) |

**과천과학관 직무와의 연결 지점**

- 본 프로젝트는 **"과학 상식 질문에 대해 관련 문서를 정확히 찾아오는 시스템"** 을 설계·구축한 것으로, 과학탐구과의 **AI(Notebook LM 등) 기반 전시해설 서비스 개발** 업무 구조와 동일합니다.
- 비지식·스몰토크 질의를 걸러내는 **Smalltalk Guard** 를 도입한 경험은, 전시해설 서비스에서 **관람객의 질문이 전시 범위를 벗어날 때 어떻게 응답할지** 를 설계하는 데 그대로 이식 가능합니다.
- Elasticsearch 기반 색인·검색 경험은, 과천과학관 내 방대한 전시 자료를 **검색 가능한 지식베이스로 정제**하는 작업에 직접 적용됩니다.

### 6.2 [LLM] 정책지원금 Agent AI

> 정책 공고 **508건**을 구조화하고, **LangGraph StateGraph** 로 Search / QA / Eligibility Agent를 분리한 뒤 **근거 충분성 노드 · 웹 검색 폴백** 으로 환각을 구조적으로 억제한 RAG + Agent 서비스.

| 항목 | 내용 |
|---|---|
| 기술 스택 | Next.js (UI) · FastAPI · **Qdrant** (벡터 DB) · MySQL · BGE-m3 임베딩 · **LangGraph** · LangSmith (트레이싱) · OpenAI API · Docker Compose |
| 핵심 기능 | LangGraph 기반 Agent 분리 · 벡터 검색 + 캐시 · 근거 조항 인용 · Tool 기반 웹 검색 폴백 · 멀티턴 자격 판정(`ELIGIBLE` / `NOT_ELIGIBLE` / `CANNOT_DETERMINE`) |
| 성과 | 캐시·하이브리드 검색·Agent 분리로 **응답 속도 대폭 개선** (프로젝트 기준 약 100배 수준) |

**배운 것 (과천과학관 전시해설 업무와의 연결점)**

- 모델을 교체하는 것보다 **문서를 어떻게 구조화·정제하는가**가 결과물 신뢰도를 좌우한다는 점을 체감.
- **근거 조항을 답변에 함께 노출하는 UI** 설계 경험은, 과천과학관 AI 전시해설 서비스에서 **"어떤 전시 패널의 어느 부분에서 나온 설명인지"** 를 투명하게 보여주는 인터페이스에 활용 가능.

### 6.3 [MLOps] 게임 추천 웹페이지 배포 ― 팀장 경험

## ⭐ 7. 과천과학관 AI 전시해설 서비스 가상 제안

> 채용공고 "AI(Notebook LM 등) 기반 전시해설 서비스 개발" 업무를 가정하고, §6.1 IR 프로젝트 · §6.2 LLM Agent 프로젝트 구조를 과천과학관 환경에 이식할 경우의 1차 설계안입니다.

### 7.1 제안 아키텍처

```
관람객 질문
    │
    ▼
[Smalltalk Guard] ──── (전시 범위 외 질의는 별도 응답 템플릿)
    │
    ▼
[Hybrid Retrieval]
  ├─ Sparse : 전시 패널 텍스트 BM25 인덱스
  └─ Dense  : 패널·관련 논문·공공 자료 임베딩 (Qdrant 또는 Elasticsearch)
    │
    ▼
[RRF 통합 → 2-stage Reranking]
    │
    ▼
[LLM Agent (근거 충분성 노드)]
  ├─ 근거 충분 → 인용 각주 포함 답변 생성
  └─ 근거 부족 → "추가 자료 필요" 안내 / 관람 동선 추천
    │
    ▼
관람객 응답 (출처 패널 번호 + 본문)
```

### 7.2 연령별 응답 스타일 분기 (설계 예시)

| 관람객 | 질문 예시 | 답변 스타일 |
|---|---|---|
| 초등 저학년 | "DNA는 왜 두 줄이야?" | 3~4문장, 비유 중심, "궁금하면 패널 ⑦ 에 설명이 더 있어요" |
| 중·고등 | "반보존적 복제가 뭐예요?" | 핵심 개념 + 패널 번호 인용, 전시물 위치 안내 |
| 성인 | "이 전시 내용의 출처 논문이 궁금합니다" | 전시 패널 + 원문 논문 링크, 필요 시 관련 전시실로 동선 제안 |

### 7.3 환각·오안내 방지 가드레일

- **근거 충분성 노드** : 문서 검색 결과의 신뢰도가 낮으면 답변 생성을 보류하고 사람 안내원 연결을 권유.
- **출처 명시 UI** : 답변에 "패널 번호 · 섹션" 을 항상 함께 노출.
- **Smalltalk Guard** : 전시와 무관한 잡담성 질의에는 별도 응답 템플릿을 사용해, 전시 콘텐츠에 대한 잘못된 설명이 생성되지 않도록 방어.

> 🛠 **기술적 실행 가능성** : §6.1 IR 프로젝트에서 Elasticsearch · KURE-v1 · BGE-reranker 앙상블을 이미 구축해 본 경험이 있고, §6.2 LLM Agent 프로젝트에서 LangGraph · 근거 충분성 노드를 직접 설계해 봤습니다. 위 제안은 해당 경험을 **과천과학관 환경(전시 패널 · 관람객 · 동선)** 에 맞춰 재구성한 것으로, 청년인턴 기간 내 프로토타입 수준의 구현이 가능한 범위입니다.
